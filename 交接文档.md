## 一、项目概述

| 项目信息 | 详情 |
|----------|------|
| **项目名称** | EcoWise 宿舍助理 |
| **所属赛道** | 物联网竞赛 - 涂鸦赛道 |
| **项目定位** | 宿舍智能用电管理系统 |
| **技术栈** | Python + Flask + 涂鸦云 API + SQLite + 通义千问 AI + Chart.js |
| **部署方式** | Flask 网页版（替代小程序方案），已部署至 Render |
| **文件路径** | `D:\VS\code\Ecowise\` |
| **运行端口** | 5000（开发环境），Render自动分配（生产环境） |
| **生产地址** | `https://ecowise-c87k.onrender.com` |

### 核心目标
- 实时监测宿舍用电情况（功率、电压、电流）
- 智能识别违规大功率电器（功率>800W持续5分钟）
- 自动断电保护功能（冷却期10分钟）
- AI 生成用电周报和智能问答
- 多用户注册与设备管理
- 碳排放计算与展示
- 作息分析与熬夜提醒（已移除UI入口）
- 设备配对机制（插座+开发板传感器数据融合）
- 浏览器推送通知

## 五、API 接口详细说明

### 基础信息
- **基础URL**: `http://localhost:5000`（开发）/ `https://ecowise-c87k.onrender.com`（生产）
- **认证方式**: Session Cookie（前端请求需添加 `credentials: 'include'`）
- **响应格式**: JSON `{ "success": true/false, "data": {...} }`
- **CORS配置**: 动态Origin + Access-Control-Allow-Credentials: true

## 七、核心业务逻辑

### 7.1 违规检测流程

```
设备数据轮询(每10秒)
       │
       ▼
┌───────────────────┐
│  check_and_record │
│   (violation_     │
│    detector.py)   │
└────────┬──────────┘
         │
    功率 > 800W?
         │
    ┌────┴────┐
    │         │
   是        否
    │         │
    ▼         ▼
 开启/继续  结束违规事件
 违规事件    │
    │    持续时间 >= 5分钟?
    │         │
    │    ┌────┴────┐
    │    │         │
    │   是        否
    │    │         │
    │    ▼         ▼
    │  确认违规   删除事件
    │  发送通知   (丢弃)
    │  触发断电
    │
    ▼
 记录到 violation_events
```

### 7.2 自动断电流程

```
违规确认
    │
    ▼
trigger_power_off()
    │
    ├─► control_device_switch(device_id, False)
    │         │
    │         └── 调用涂鸦云API关闭插座
    │
    ├─► 写入 power_off_events
    │         │
    │         └── is_released=0, cooldown_until=now+10分钟
    │
    └─► 发送通知给设备owner
```

### 7.3 冷却期管理

```
用户点击"开启插座"
    │
    ▼
检查 is_in_cooldown(device_id)
    │
    ┌───────┬───────┐
    │       │       │
 冷却中   已过期   无事件
    │       │       │
    ▼       ▼       ▼
拒绝开启  release_event()  正常开启
          标记is_released=1
```

### 7.4 设备配对与数据融合

```
设备列表加载
    │
    └─► 遍历所有设备
         │
         ├─► 标记 is_sensor（开发板/传感器）
         │
         └─► 检查 DEVICE_PAIRINGS
              │
              ├─► 插座设备 ←─── 查找配对的传感器设备
              │
              └─► 合并传感器数据（温湿度）到插座设备
                   │
                   ▼
              前端显示插座+传感器融合数据
```

### 7.5 历史数据采集

```
定时任务(每10分钟)
       │
       ▼
 record_all()
       │
       └─► 遍历所有设备
           │
           └─► get_device_data(device_id)
               │
               └─► INSERT INTO energy_records
                   (energy_wh, power_w, light_level, ...)
```

### 7.6 AI对话历史管理

```
用户发送消息
    │
    ├─► 前端保存到 aiChatHistory 数组
    │
    └─► 发送请求到 /api/ai
         │
         ├─► 携带 history 参数
         │
         └─► 后端注入历史到通义千问上下文
              │
              └─► 返回回复
                   │
                   └─► 前端追加到 aiChatHistory
```

## 十二、常见问题与解决方案

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `ModuleNotFoundError: flask` | Python环境缺少依赖 | `pip install -r requirements.txt` |
| `PermissionError` 写入文件失败 | 项目目录无写权限 | 数据已移至 `%TEMP%\Ecowise\` |
| 网页显示"加载中"一直转圈 | API请求失败或异常 | 检查服务器日志，确认后端运行正常 |
| 端口5000被占用 | 旧进程未关闭 | `taskkill /F /PID <进程号>` |
| 功率显示错误（20W显示200W） | 单位换算系数错误 | 确认 `POWER_RAW_UNIT_W = 0.1` |
| 手机无法访问 | 网络不通 | 确认电脑和手机同一Wi-Fi，关闭防火墙 |
| 添加设备提示"已被添加过" | 设备已在DEVICES中 | 修改添加逻辑，允许覆盖默认设备 |
| 测试模式下违规记录不显示 | 查询逻辑限制了已确认记录 | 已修复，显示所有状态记录 |
| 图表无法左右滑动 | Chart.js配置问题 | 已修复，`drawChart` 支持 `enableScroll` 参数 |
| 手机浏览器登录无反应 | 跨域CORS配置问题 | 已修复：动态Origin + credentials |
| 注册验证码计时器不重置 | 切换表单时未清除计时器 | 已修复：新增 `resetVerifyCodeButton()` |
| AI对话不连贯 | 未维护对话历史 | 已修复：前端保存历史并传给后端 |
| 设备配对分类错误 | 分类逻辑仅根据group判断 | 已修复：新增 `is_sensor` 字段 |
| 电费分摊显示0.001度 | 精度问题 | 已修复：MIN_KWH_THRESHOLD=0.01 |
| 测试模式退出后数据未清除 | 缺少清除逻辑 | 已修复：`clear_today_records()` |
| 服务器重启后登录状态未清除 | 测试模式未清空用户表 | 已修复：添加用户表清除逻辑 |
| 柱状图数字被遮挡 | 顶部padding不足 | 已修复：padding.top从25增加到45 |
| 开发板数据不显示 | 虚拟设备ID或DP点未上传 | 使用真实设备ID，检查DP点上传状态 |

## 十五、后续维护建议

### 高优先级
1. **解决手机端登录问题**: 确认JS错误捕获输出，修复跨域/session问题
2. **完整功能测试**: 在Render部署后重新测试所有修改点
3. **硬件数据验证**: 确认插座和开发板的真实数据能正确显示
4. **考虑国内部署**: 如果Render访问不稳定，可考虑部署到国内云平台（如阿里云、腾讯云）

### 中优先级
1. **数据库持久化**: 生产环境确认 `CLEAR_DATA_ON_START=false` 生效
2. **WebSocket实时推送**: 使用Flask-SocketIO替代轮询，降低服务器负载
3. **HTTPS证书**: Render自动配置SSL，如需自定义域名需单独配置
4. **性能监控**: 添加前端性能监控和错误上报

### 低优先级
1. **UI进一步美化**: 根据用户反馈持续优化
2. **密码加密升级**: 当前使用MD5，建议升级为bcrypt
3. **API限流**: 添加请求频率限制防止滥用
4. **数据导出**: 支持导出Excel格式的用电报表
