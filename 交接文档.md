# EcoWise 宿舍助理 - 项目交接文档

---

## 一、项目概述

| 项目信息 | 详情 |
|----------|------|
| **项目名称** | EcoWise 宿舍助理 |
| **所属赛道** | 物联网竞赛 - 涂鸦赛道 |
| **项目定位** | 宿舍智能用电管理系统 |
| **技术栈** | Python + Flask + 涂鸦云 API + SQLite + 通义千问 AI |
| **部署方式** | Flask 网页版（替代小程序方案） |
| **文件路径** | `D:\VS\code\Ecowise\` |
| **运行端口** | 5000 |

### 核心目标
- 实时监测宿舍用电情况（功率、电压、电流）
- 智能识别违规大功率电器（功率>800W持续5分钟）
- 自动断电保护功能（冷却期10分钟）
- AI 生成用电周报和习惯分析
- 多用户注册与设备管理
- 碳排放计算与展示
- 作息分析与熬夜提醒

## 七、核心业务逻辑

### 7.1 违规检测流程

```
设备数据轮询(每10秒)
       │
       ▼
┌───────────────────┐
│  check_and_record │
│   (violation_     │
│    detector.py)   │
└────────┬──────────┘
         │
    功率 > 800W?
         │
    ┌────┴────┐
    │         │
   是        否
    │         │
    ▼         ▼
 开启/继续  结束违规事件
 违规事件    │
    │    持续时间 >= 5分钟?
    │         │
    │    ┌────┴────┐
    │    │         │
    │   是        否
    │    │         │
    │    ▼         ▼
    │  确认违规   删除事件
    │  发送通知   (丢弃)
    │  触发断电
    │
    ▼
 记录到 violation_events
```

### 7.2 自动断电流程

```
违规确认
    │
    ▼
trigger_power_off()
    │
    ├─► control_device_switch(device_id, False)
    │         │
    │         └── 调用涂鸦云API关闭插座
    │
    ├─► 写入 power_off_events
    │         │
    │         └── is_released=0, cooldown_until=now+10分钟
    │
    └─► 发送通知给设备owner
```

### 7.3 冷却期管理

```
用户点击"开启插座"
    │
    ▼
检查 is_in_cooldown(device_id)
    │
    ┌───────┬───────┐
    │       │       │
 冷却中   已过期   无事件
    │       │       │
    ▼       ▼       ▼
拒绝开启  release_event()  正常开启
          标记is_released=1
```

### 7.4 历史数据采集

```
定时任务(每10分钟)
       │
       ▼
 record_all()
       │
       └─► 遍历所有设备
           │
           └─► get_device_data(device_id)
               │
               └─► INSERT INTO energy_records
                   (energy_wh, power_w, light_level, ...)
```

### 测试模式说明

当前项目处于测试阶段，`web_server.py` 中设置了 `CLEAR_DATA_ON_START = True`，每次启动会自动：
1. 清空 `config.DEVICES`（包括默认设备）
2. 删除 `devices.json`
3. 删除数据库文件

**确认功能无误后**，将 `CLEAR_DATA_ON_START` 改为 `False` 即可保留数据。

## 十、测试模式使用说明

### 开启测试模式

1. 在实时监控页面找到"测试模式"区域
2. 勾选"开启测试模式"开关
3. 调整功率滑块（建议设置为 1200W 以触发违规断电）
4. 点击保存

### 测试场景

| 功率设置 | 效果 |
|----------|------|
| 500W | 触发限电预警（warning） |
| 1200W | 触发违规告警（violation），持续5分钟后自动断电 |
| 700W | 正常用电范围 |

### 注意事项

- 测试模式下的数据为模拟数据，不会产生真实电费
- 违规事件记录会保存到数据库，可在违规记录中查看
- 关闭测试模式后，需要手动清除断电记录（已自动处理）

## 十一、常见问题与解决方案

| 问题 | 原因 | 解决方法 |
|------|------|----------|
| `ModuleNotFoundError: flask` | Python环境缺少依赖 | `pip install flask requests` |
| `PermissionError` 写入文件失败 | 项目目录无写权限 | 数据已移至 `%TEMP%\Ecowise\` |
| 网页显示"加载中"一直转圈 | API请求失败或异常 | 检查服务器日志，确认后端运行正常 |
| 端口5000被占用 | 旧进程未关闭 | `taskkill /F /PID <进程号>` |
| 功率显示错误（20W显示200W） | 单位换算系数错误 | 确认 `POWER_RAW_UNIT_W = 0.1` |
| 手机无法访问 | 网络不通 | 确认电脑和手机同一Wi-Fi，关闭防火墙 |
| 添加设备提示"已被添加过" | 设备已在DEVICES中 | 检查 `config.py` 中的默认设备是否需要清空 |
| 测试模式下违规记录不显示 | 查询逻辑限制了已确认记录 | 已修复，`get_today_violations` 显示所有状态记录 |
| 图表无法左右滑动 | Chart.js配置问题 | 已修复，`drawChart` 支持 `enableScroll` 参数 |

## 十三、后续维护建议

### 功能优化
1. **WebSocket实时推送**: 使用Flask-SocketIO替代轮询，降低服务器负载
2. **告警通知**: 违规时通过微信公众号/短信推送通知
3. **数据导出**: 支持导出Excel格式的用电报表
4. **设备指纹识别优化**: 接入更精准的电器识别算法

### 安全优化
1. **密码加密**: 当前使用MD5，建议升级为bcrypt
2. **HTTPS支持**: 部署到公网时启用HTTPS
3. **请求频率限制**: 添加API限流防止滥用
4. **SQL注入防护**: 使用参数化查询（已实现）

### 性能优化
1. **数据库索引**: 为频繁查询的字段添加索引（已实现）
2. **数据缓存**: 对设备数据添加缓存，减少涂鸦云API调用
3. **异步任务**: 周报生成等耗时操作改为异步执行

### Bug修复优先级
1. **通知系统**: 确保违规通知能正确发送到前端
2. **测试模式**: 确保所有功能在测试模式下正常工作
3. **图表渲染**: 确保所有图表显示正常，无遮挡
